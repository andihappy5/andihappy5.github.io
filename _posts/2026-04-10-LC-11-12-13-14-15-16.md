# 题目描述  

11. Container With Most Water   

You are given an integer array height of length n. There are n vertical lines drawn such that the two endpoints of the ith line are (i, 0) and (i, height[i]).

Find two lines that together with the x-axis form a container, such that the container contains the most water.

Return the maximum amount of water a container can store.

Notice that you may not slant the container.

 

Example 1:


Input: height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation: The above vertical lines are represented by array [1,8,6,2,5,4,8,3,7]. In this case, the max area of water (blue section) the container can contain is 49.
Example 2:

Input: height = [1,1]
Output: 1
 

Constraints:

n == height.length
2 <= n <= 105
0 <= height[i] <= 104

## code
~~~java
    public static int maxArea(int[] height) {
        int res = 0;
        int left = 0;
        int right = height.length - 1;
        while (left < right) {
            int min = Math.min(height[left], height[right]);
            res = Math.max(res, min * (right - left));
            while (left < right && height[left] <= min)
                left++;
            while (left < right && height[right] <= min)
                right--;
        }
        return res;
    }
~~~
## explanation in interview 
the question is to find the maximum area of water that can be contained by two lines in the given array. 

My approach is to use a two-pointer technique, where I start with one pointer at the beginning of the array and another pointer at the end of the array. I will calculate the area formed by the lines at these two pointers and when I calculate the area, I will update the maximum area if the current area is greater than the previously recorded maximum area. 

Then, I will move the pointer that points to the shorter line towards the other pointer, because moving the pointer that points to the taller line will not increase the area. I will repeat this process until the two pointers meet.

Time complexity is O(n) where n is the length of the input array, because we may need to process each element in the worst case. Space complexity is O(1) since we are using a constant amount of space for the variables.

## code
~~~java
public int threeSumClosest(int[] nums, int target) {
            // first sort the array
            Arrays.sort(nums);
            int closest = Integer.MAX_VALUE;
            int result = 0;
            for (int i = 0; i < nums.length - 2; i++) {
                if (i > 0 && nums[i] == nums[i - 1]) {
                    continue;
                }
                int left = i + 1;
                int right = nums.length - 1;
                while (left < right) {
                    int sum = nums[i] + nums[left] + nums[right];
                    if (Math.abs(sum - target) < Math.abs(closest)) {
                        closest = Math.abs(sum - target);
                        result = sum;
                    }
                    if (sum - target == 0) {
                        return target;
                    } else if (sum - target < 0) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
            return result;
        }
~~~

## explanation in interview 
The question is to find the sum of three integers in the given array that is closest to the target value.

My approach is to first sort the array, and then use a two-pointer technique to find the closest sum. I will iterate through the array with a loop, and for each element, I will set two pointers: one starting from the next element and another starting from the end of the array. I will calculate the sum of the three elements and compare it with the target. If the absolute difference between the sum and the target is smaller than the previously recorded closest difference, I will update the closest difference and store the current sum as the result. If the sum is equal to the target, I can return the target immediately since it is the closest possible. If the sum is less than the target, I will move the left pointer to the right to increase the sum, and if the sum is greater than the target, I will move the right pointer to the left to decrease the sum. I will repeat this process until the two pointers meet.

Time complexity is O(n^2) where n is the length of the input array, because we have a loop that iterates through the array and for each element, we have a two-pointer technique that can take O(n) time in the worst case. Space complexity is O(1) since we are using a constant amount of space for the variables.
